Примитивные типы данных

Null

Тип данных определен, но содержит "НИЧЕГО", "ПУСТО" или "ЗНАЧЕНИЕ НЕ ОПРЕДЕЛЕНО"

Undefined

Тип данных не определен

Number

Значением может быть:

  • Целое число: 123
  • Число с плавающей точкой: 1.23
  • Результать деления на 0: Infinity
  • Результат деления на 0 отрицательного числа: - Infinity
  • Математическая неудачаА: NaN. ("Not a Number", другими словами: результат вычислений не может быть представлен как число)

BigInt

Ну ооооощень балшой число!

123123123123n

Больше чем 253-1, или меньше чем -253-1

String

Данные в текстовом представлении. Встречаются в разных кавычках.

  1. " " и '' - равноценны
  2. `` - обратные кавычка(бывают и такие, ага) позволяют обращаться к переменной внутри данной строки

Смотри-ка примерчик подвезли!
`User ${user.name} is ${user.age} years old`

Boolean

Логический тип. Может только в правду или ложь.
Но, что характерно, вернется как тип для такой вот, к примеру, вычисляемой переменной:
let someVar = 13 > 1;

Symbol

Через функцию Symbol() создает уникальное и неизменяемое значение. Сразу видно штука нужная, потом пойму.

Не примитивные типы данных

Object

Object - не примитив, потому что может содержать в себе больше одного значения одновременно. И даже больше одного объекта, и скорее всего силу Земли...

Говорят, что функции и метододы в JS тоже немного Object...

Создание объекта

С помощью конструктора:
let user = new Object();
Применяется, когда нужно создать много однотипных объектов.

С помощью литеральной нотации:
let user = {};
Можно сразу задать свойства объекта. Свойства это ключ(имя) : значение ,

Запись свойств объекта

Разные способы задания свойств объекта:

let oneMoreKey = "color";
let user = {
name: "CatInHat",
age: 13,
"food type": "meat",
[oneMoreKey]: "black",
};

Добавление и удаление свойств:
user.favoriteSpot = {
day: "kitchen",
night: "bed",
};

delete user[ ' name ' ];

Разрешите обратиться! или обращение к свойствам объекта

Можно обращаться через точку . или с помощью квадратных скобок [ ]. Например:

console.log(user.name);

console.log(user["name"]);

К свойствам с пробелом в имени типа "food type" можно обращаться ТОЛЬКО через квадратные скобки, но есть мнение, что такой способ имеет преимущества, например, возможноть реализовать подобное обращение через переменную:

let key = "name";
console.log(user[key]);

А напоследок я скажу...

Еще мы узнали, что запись типа user = userBot не копирует объект, а создает ссылку на него(мне нравится аналогия с дубликатом ключей к камере хранения).

Для создания клона используется метод Object.assign().

Функции и Методы

Функция?

Функция - объект, позволяющий реализовывать определенную логику.... бла-бла-бла... умные слова... Функция что-то делает с объектом или создавать объект(ы), может возвращать что-то в результате выполнения или не возвращать. Такая она...

Метод!

Мне нагляднее всего показалось такое описание: "обычное свойство объекта хранит значение, а метод хранит функцию" т.е. метод - это функция заранее прописанная(уже в стандарте JS?) под конкретный тип объекта.
